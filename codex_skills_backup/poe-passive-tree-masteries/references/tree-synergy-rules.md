# Passive Tree Synergy Rules

Record passive relationships by role:
- damage
- defense
- sustain
- reservation
- charges
- crit
- ailments
- minions
- projectiles
- duration
- AoE
- pathing
- jewel_socket
- cluster

Each record needs node name/ID if available, reason, class/ascendancy context, patch, source, and confidence.

Never mix passive tree data across patches. If node ID is unknown, keep name plus `id_status: unverified`.
