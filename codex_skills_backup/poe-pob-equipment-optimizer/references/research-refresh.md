# Research Refresh

Use this for the recurring 6-hour learning job or manual refreshes.

## Purpose

Keep the PoE 1 item optimizer aligned with current project data and recent public knowledge without allowing web text to override PoB/RePoE/GGG structural data.

## Categories To Refresh

- PoB item XML format and import behavior.
- RePoE/static data availability.
- Rare affix pools and item level legality.
- Bench, essence, fossil, eldritch, influenced, fractured and corrupted crafting states.
- Unique item and unique jewel exact behavior.
- Cluster jewel bases and notables.
- Timeless jewel and radius jewel behavior.
- Flask affixes and uptime assumptions.
- Meta/budget evidence from poe.ninja, treated only as market/meta evidence.

## Source Priority

1. Local project data and tests.
2. Path of Building source/data.
3. GGG static exports or official game data.
4. RePoE/forked static exports.
5. poe.ninja market/meta data.
6. Community explanations only for human notes.

## Refresh Output

For each refresh, update project docs or create a dated note only when there is concrete new evidence. Include:

- source URL or local file;
- category;
- finding;
- project impact;
- whether code/data should change;
- blocker code if relevant.

Never silently promote web claims into the optimizer. Structural truth must become local data, a test, or a documented blocker.
